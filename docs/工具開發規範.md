## 1. 工具基本結構

```
tools/your-tool-name/
├── manifest.json   (必要)
├── icon.png        (選用，建議 48×48)
├── index.html      (必要)
└── config.js       (選用，第三方 API 設定等)
```

### manifest.json 範例

```json
{
  "name": "工具顯示名稱",
  "icon": "icon.png",
  "entry": "index.html",
  "version": "1.0.0",
  "description": "簡短描述",
  "category": ["image", "workflow"],
  "minPsVersion": "20.0"
}
```

## 2. 與主框架溝通

工具運行於 iframe，透過 `postMessage` 與主框架溝通。

### 呼叫 PS API

```javascript
function callPsApi(action, params) {
  return new Promise(function (resolve, reject) {
    var callId = 'call_' + (++callCounter);
    pendingCalls[callId] = { resolve: resolve, reject: reject };
    window.parent.postMessage({
      type: 'ps-api', action: action, params: params || {}, callId: callId
    }, '*');
    setTimeout(function () {
      if (pendingCalls[callId]) { delete pendingCalls[callId]; reject('Timeout'); }
    }, 60000);
  });
}
```

可用的 API 列表請見 `js/ps-bridge.js`。

## 3. ⚠️ 長任務必須回報忙碌狀態

> **這是強制規範。** 如果工具有任務超過 3 秒以上才會結束（API 呼叫、批次匯出、雲端運算等），**一定要回報忙碌狀態**，否則使用者按返回會直接砍掉 iframe，任務會在無預警情況下死亡。

### 為什麼？

主框架的 `goHome()` 會把 iframe 設為 `about:blank`，**任何進行中的 async 任務（fetch、polling、長時間 ExtendScript 呼叫）會立即被中斷**。如果工具有回報 busy state，主框架會在使用者按返回時跳出確認框。

### 規範用法

```javascript
// 任務開始時通知忙碌
window.parent.postMessage({
  type: 'set-busy',
  busy: true,
  message: '正在執行 XXX 任務（預計 OO 分鐘）'
}, '*');

try {
  // ... 你的長任務 ...
} finally {
  // 任務結束（不論成功 / 失敗 / 取消）一定要清除
  window.parent.postMessage({ type: 'set-busy', busy: false }, '*');
}
```

### 訊息文字建議

- 簡短一句話描述「正在做什麼」
- 加上預期時間，讓使用者判斷要不要等
- 範例：
  - `'SeedVR2 放大任務進行中（雲端運算可能耗時數分鐘）'`
  - `'UI 匯出進行中（每張圖層需要 PS 處理時間）'`
  - `'正在處理 12 / 50 張圖片…'`（動態更新可重複呼叫）

### 主框架的行為

| 觸發 | 行為 |
|------|------|
| busy=true 時使用者按「← 返回」 | 跳 confirm 警告 + 你提供的 message，使用者可選 OK 或 Cancel |
| busy=true 時使用者切換到別的工具 | （目前沒攔，因為 openTool 會直接重設 iframe） |
| iframe 被銷毀 | busy state 自動清除（state.toolBusy 重置） |

## 4. 取消按鈕設計

提供取消按鈕讓使用者中止長任務。建議做法：

### 4.1 用旗標 + 可取消的 sleep

```javascript
var cancelRequested = false;

// 可取消的 sleep
var _sleepRejectors = [];
function sleep(ms) {
  return new Promise(function (resolve, reject) {
    var rejector = function () { reject(new Error('已取消')); };
    var timer = setTimeout(function () {
      var idx = _sleepRejectors.indexOf(rejector);
      if (idx >= 0) _sleepRejectors.splice(idx, 1);
      resolve();
    }, ms);
    rejector._timer = timer;
    _sleepRejectors.push(rejector);
  });
}
function abortAllSleeps() {
  while (_sleepRejectors.length > 0) {
    var r = _sleepRejectors.pop();
    try { clearTimeout(r._timer); } catch(e) {}
    try { r(); } catch(e) {}
  }
}

// 取消按鈕處理
function cancelTask() {
  cancelRequested = true;
  abortAllSleeps();  // 立即跳出 await sleep
  // 視需要呼叫雲端 API 通知取消（fire-and-forget，不要 await）
}

// 任務迴圈
async function runTask() {
  for (...) {
    if (cancelRequested) throw new Error('已取消');
    // ... do work ...
    await sleep(2000);
  }
}
```

### 4.2 注意事項

- **取消按鈕的 fetch 用 fire-and-forget**：不要 await，否則會卡住 UI
- **同步先設旗標 + UI 反饋**：使用者按下後立即看到「取消中…」，不等 await
- **每個 await 後加 checkCancel()**：確保盡快響應，不會做完無謂的步驟

## 5. 狀態保存（離開後回來能看到）

由於 iframe 會被砍掉，**JS 變數狀態都會消失**。如果想在使用者離開回來時看到 log/結果，請用 `localStorage`。

```javascript
var STORAGE_KEY = 'mytool_state_v1';

function saveState(s) { localStorage.setItem(STORAGE_KEY, JSON.stringify(s || {})); }
function loadState() {
  try { return JSON.parse(localStorage.getItem(STORAGE_KEY) || '{}'); }
  catch(e) { return {}; }
}
```

### 注意

- localStorage 容量有限（~5MB），不要存大型二進位資料
- 加版本號 `_v1`，未來資料結構改變時容易管理
- 加上「清除」按鈕，讓使用者能手動清空

### 「殭屍狀態」處理

若上次任務被中斷（iframe 死亡）但 localStorage 還顯示在執行中，回到工具時應該偵測並標記為「已中斷」，否則使用者會看到一個假裝在跑但其實死掉的 UI。

```javascript
function restoreState() {
  var st = loadState();
  if (st.lastProgress > 0 && st.lastProgress < 100 && !st.completed) {
    // 上次未完成 → 標記為中斷
    addLog('━━━ 上次執行已被中斷 ━━━', 'warn');
    setProgress(0);
    setStatus('上次任務已中斷，請重新執行', 'error');
  }
}
```

## 6. 第三方 API 設定（config.js 模式）

對於需要 API key、雲端 endpoint 的工具，建議獨立放在 `config.js`：

```javascript
// config.js
window.MYTOOL_CONFIG = {
  API_KEY: "xxxxxxxx",
  BASE_URL: "https://api.example.com",
  // ...
};
```

```html
<!-- index.html -->
<script src="./config.js"></script>
<script>
  var CFG = window.MYTOOL_CONFIG;
  // ...
</script>
```

好處：
- 換 API 帳號 / 端點時只改一個檔案
- 不用碰 index.html 主程式邏輯
- 開發 / 生產環境切換簡單

## 7. UI 設計建議

- 沿用框架的色系（在工具裡複製主面板的 CSS variables）
- 暗色主題（避免在 PS 暗介面下刺眼）
- 字體：`"Adobe Clean", "Segoe UI", sans-serif`
- 按鈕風格：圓角 4px，hover 略微提亮 / 上浮
- 進度條、log 視覺風格參考 `tools/seedvr2-upscale/index.html` 的範例

## 8. 上線檢查清單

提交工具前確認：

- [ ] `manifest.json` 欄位完整、版本號正確
- [ ] 長任務有回報 `set-busy`（開始 + 結束兩端都要）
- [ ] 取消按鈕用 `abortAllSleeps()` 即時中斷
- [ ] 重要狀態有存 localStorage（可選）
- [ ] 偵測殭屍狀態（任務未完成的警告）
- [ ] 錯誤處理完整（try/catch 包住主流程）
- [ ] UI 在不同 PS 視窗大小下都不會破版（最小 300×400）
- [ ] 中文文字無亂碼（建議統一用繁體 zh-TW）
